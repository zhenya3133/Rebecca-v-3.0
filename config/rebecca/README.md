# Конфигурация мета-агента Rebecca

Данная директория содержит полный набор конфигурационных файлов для мета-агента Rebecca - системы оркестрации специализированных агентов.

## Структура файлов

### Основные конфигурационные файлы

1. **`agent_config.yaml`** - Основная конфигурация агента
   - Настройки мета-агента (timeouts, retries, limits)
   - Конфигурация специализированных агентов (backend, frontend, ml, qa, devops)
   - Настройки планирования и приоритизации
   - Конфигурация системы памяти
   - Настройки безопасности и интеграций

2. **`prompt_templates.yaml`** - Шаблоны промптов
   - Системные промпты для мета-агента
   - Промпты для планирования задач
   - Шаблоны для generate_playbook()
   - Инструкции для каждого типа подагента
   - Обработка исключительных ситуаций
   - Шаблоны валидации и контроля качества

3. **`workflow_templates.yaml`** - Шаблоны workflow
   - Стандартные workflow шаблоны (basic, web_development, ml_project, api_development)
   - Последовательности выполнения для разных типов задач
   - Условия для переходов между шагами
   - Fallback сценарии
   - Параллельные и последовательные паттерны
   - Настройки мониторинга и контроля

4. **`__init__.py`** - Python модуль конфигурации
   - Класс `RebeccaConfig` для централизованного управления
   - Функции-утилиты для загрузки и валидации
   - Поддержка экспорта/импорта конфигураций

### Примеры конфигураций

Папка `examples/` содержит готовые конфигурации для различных сценариев:

1. **`startup_config.yaml`** - Конфигурация для стартапов
   - Быстрая разработка с ограниченными ресурсами
   - Агрессивные таймауты и минимальные лимиты
   - Упрощенные настройки безопасности
   - Фокус на MVP и прототипирование

2. **`enterprise_config.yaml`** - Конфигурация для крупных предприятий
   - Высокие требования к качеству и масштабируемости
   - Продвинутые настройки безопасности и комплаенса
   - Корпоративные интеграции
   - Высокая доступность и disaster recovery

3. **`research_config.yaml`** - Конфигурация для исследовательских проектов
   - Поддержка долгих экспериментов
   - Интеграция с научными инструментами
   - Фокус на воспроизводимость и публикацию
   - Экспериментальные функции

4. **`education_config.yaml`** - Конфигурация для образовательных платформ
   - Адаптивное обучение и персонализация
   - Интеграция с LMS системами
   - Фокус на доступность и вовлеченность
   - Безопасность студенческих данных

## Быстрый старт

### Загрузка конфигурации

```python
from config.rebecca import load_rebecca_config, RebeccaConfig

# Загрузка конфигурации по умолчанию
config = load_rebecca_config()

# Или загрузка с указанием директории
config = RebeccaConfig("/path/to/config")

# Проверка валидности конфигурации
is_valid, errors = config.validate_configuration()
if not is_valid:
    print("Ошибки конфигурации:", errors)
```

### Использование различных типов конфигураций

```python
# Загрузка конфигурации для стартапа
startup_config = RebeccaConfig("/path/to/config/examples/startup")

# Загрузка конфигурации для предприятия
enterprise_config = RebeccaConfig("/path/to/config/examples/enterprise")

# Получение настроек конкретного агента
backend_settings = config.get_agent_config("backend")
frontend_settings = config.get_agent_config("frontend")
```

### Работа с промптами

```python
# Получение системного промпта
main_prompt = config.get_prompt_template("system", "main")

# Получение промпта планирования
planning_prompt = config.get_prompt_template("planning", "task_analysis")

# Получение инструкций для агента
backend_instructions = config.get_prompt_template("agent_instruction", "backend")
```

### Использование workflow

```python
# Получение базового workflow
basic_workflow = config.get_workflow_template("basic")

# Получение workflow для веб-разработки
web_workflow = config.get_workflow_template("web_development")

# Получение ML workflow

```

## Настройки системы памяти

### Слои памяти
- **working_memory**: Краткосрочная память для текущих операций
- **episodic_memory**: Память о выполненных задачах и результатах
- **semantic_memory**: Семантическая память с знаниями и концепциями
- **procedural_memory**: Процедурная память с алгоритмами и методами

### Кэширование
- Настраиваемый размер кэша
- Политики вытеснения (LRU, FIFO, TTL)
- Сжатие данных
- TTL для автоматической очистки

## Расширение конфигурации

### Добавление нового агента

1. Добавьте конфигурацию в `agent_config.yaml`:

```yaml
specialized_agents:
  new_agent:
    name: "NewAgent"
    capabilities:
      - capability1
      - capability2
    config:
      max_execution_time: 300
      memory_limit_mb: 512
```

2. Добавьте промпты в `prompt_templates.yaml`:

```yaml
agent_instructions:
  new_agent:
    system: "Инструкции для нового агента..."
    delegation: "Когда делегировать задачи..."
    response_format: "Формат ответа..."
```

3. Добавьте workflow шаблоны в `workflow_templates.yaml`:

```yaml
workflow_templates:
  new_agent_workflow:
    steps:
      - name: "Step 1"
        agent: "new_agent"
```
